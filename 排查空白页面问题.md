# 排查空白页面问题

## 你的情况
- ✅ GitHub Actions 工作流已成功运行
- ❌ 页面显示空白（黑屏）

## 快速排查步骤

### 步骤 1: 检查浏览器控制台错误

1. 在空白页面上，按 **F12** 打开开发者工具
2. 点击 **"Console"**（控制台）标签页
3. 查看是否有红色错误信息
4. **把错误信息截图或复制给我**

常见错误可能是：
- `Failed to load resource: 404` - 资源文件找不到
- `Uncaught SyntaxError` - JavaScript 语法错误
- `CORS error` - 跨域问题

### 步骤 2: 检查 Network 标签页

1. 在开发者工具中，点击 **"Network"**（网络）标签页
2. 刷新页面（F5）
3. 查看哪些文件加载失败（显示红色）
4. **特别关注**：
   - `index.html` 是否加载成功（200）
   - `index-xxx.js` 是否加载成功
   - `index-xxx.css` 是否加载成功

### 步骤 3: 检查 GitHub Pages 设置

1. 打开仓库 Settings → Pages
2. 确认 **Source** 选择了 **"GitHub Actions"**
3. 如果显示 "Your site is live at..."，说明设置正确

### 步骤 4: 检查工作流构建日志

1. 在 Actions 页面，点击成功的工作流运行
2. 查看 **"Build"** 步骤的日志
3. 确认 base 路径是否正确设置
4. 查看是否有构建错误

## 可能的原因和解决方法

### 原因 1: base 路径配置问题

**检查方法：**
- 查看工作流日志中 "Update vite config" 步骤的输出
- 应该显示：`Base path: /blockchain-trading-platform/`

**解决方法：**
- 如果 base 路径不正确，可能需要手动触发重新部署

### 原因 2: 资源文件路径错误

**检查方法：**
- 在浏览器控制台查看 404 错误
- 检查 Network 标签页中失败的资源

**解决方法：**
- 确认工作流正确设置了 base 路径
- 可能需要重新触发部署

### 原因 3: GitHub Pages 缓存问题

**解决方法：**
- 等待 5-10 分钟后再试
- 或者清除浏览器缓存后刷新

请先完成步骤 1 和 2，把错误信息告诉我，我会根据具体错误帮你解决。
