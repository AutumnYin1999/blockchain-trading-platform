## ⚠️ 需要确认的项目

### 1. Git 仓库状态

**检查方法（在 GitHub Desktop 中）：**
- 打开 GitHub Desktop
- 查看左侧是否显示你的项目
- 查看顶部是否显示 "Current repository: 你的项目名"

**如果还没有添加：**
- 点击 "File" → "Add Local Repository"
- 选择项目文件夹：`E:\HKBU\Trimester 2\2 Blockchain and Virtual Asset\Group work\Prototype`
- 如果提示不是 Git 仓库，点击 "create a repository"

### 2. GitHub 远程仓库

**检查方法（在 GitHub Desktop 中）：**
- 查看仓库设置（Repository → Repository Settings）
- 确认 "Remote repository" 已配置
- 应该显示类似：`https://github.com/你的用户名/你的仓库名.git`

**如果还没有创建远程仓库：**
- 在 GitHub Desktop 中点击 "Publish repository"
- 输入仓库名称
- 选择 Public（GitHub Pages 免费版需要公开）
- 点击 "Publish Repository"

### 3. GitHub Pages 设置

**检查方法（在 GitHub 网页上）：**
1. 打开你的 GitHub 仓库页面
2. 点击 "Settings" → "Pages"
3. 在 "Source" 部分：
   - ✅ 应该选择 **"GitHub Actions"**
   - ❌ 不要选择 "Deploy from a branch"

**如果还没有设置：**
- 选择 "GitHub Actions" 作为 Source
- 保存设置

### 4. 代码已推送到 GitHub

**检查方法：**
- 在 GitHub Desktop 中查看是否有未提交的更改
- 如果有，点击 "Commit to main" 提交
- 然后点击 "Push origin" 推送到 GitHub

**或者检查 GitHub 网页：**
- 打开你的仓库页面
- 应该能看到所有项目文件
- 应该能看到 `.github/workflows/deploy.yml` 文件

## 🚀 部署流程

### 步骤 1: 确保所有文件已提交并推送

在 GitHub Desktop 中：
1. 查看是否有未提交的更改（左侧显示文件列表）
2. 如果有，填写提交信息，点击 "Commit to main"
3. 点击 "Push origin" 推送到 GitHub

### 步骤 2: 触发 GitHub Actions

**自动触发：**
- 当你推送代码到 `main` 或 `master` 分支时，GitHub Actions 会自动开始部署

**手动触发：**
1. 打开 GitHub 仓库页面
2. 点击 "Actions" 标签页
3. 选择 "Deploy to GitHub Pages" 工作流
4. 点击 "Run workflow" → "Run workflow"

### 步骤 3: 等待部署完成

1. 在 "Actions" 标签页查看部署进度
2. 看到绿色的 ✅ 表示部署成功
3. 通常需要 2-5 分钟

### 步骤 4: 访问你的网站

部署完成后，访问：
- **项目页面**: `https://你的用户名.github.io/你的仓库名/`
- **个人主页**: `https://你的用户名.github.io/`

## 🔍 常见问题检查

### 问题 1: GitHub Actions 没有运行

**可能原因：**
- GitHub Pages 设置中 Source 没有选择 "GitHub Actions"
- 代码还没有推送到 GitHub

**解决方法：**
- 检查 Settings → Pages → Source 是否选择了 "GitHub Actions"
- 确保代码已推送到 GitHub

### 问题 2: 部署失败

**检查：**
- 进入 "Actions" 标签页
- 点击失败的运行记录
- 查看错误信息

**常见错误：**
- 依赖安装失败 → 检查 `package.json` 是否正确
- 构建失败 → 检查代码是否有错误
- 权限问题 → 检查仓库 Settings → Actions → General → Workflow permissions

### 问题 3: 页面显示空白

**可能原因：**
- `base` 路径配置不正确

**解决方法：**
- 工作流会自动配置 base 路径
- 如果仍有问题，检查构建日志中的 base 路径
